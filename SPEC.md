# Polymarket BTC 5m Bot - 项目规格文档

## 1. 项目概述

- **项目名称**: polymarket-btc5m-bot
- **项目类型**: 自动化交易机器人 (本地运行)
- **核心功能**: 在 Polymarket 的 BTC-updown-5m 市场进行自动化交易
- **目标用户**: 交易者（本地使用，不推荐线上部署）

## 2. 交易逻辑规格

### 2.1 订单执行时机
- **触发时间**: 每个 5 分钟周期的第一分钟内（即 0:00 - 1:00）
- **执行操作**: 同时下限价单买入 Yes 和 No
- **价格**: 10 (代币价格)
- **数量**: 0.5

### 2.2 订单管理
- **监控期**: 每个 5 分钟周期的最后 30 秒（即 4:30 - 5:00）
- **成交检测**: 检查 Yes 和 No 订单状态
- **取消逻辑**: 如果其中一个订单已成交，取消另一个未成交的订单
- **循环**: 持续监控直到交易停止

### 2.3 交易 API
- 使用 Polymarket GraphQL API 进行交易
- 需要私钥签名交易

## 3. UI/UX 规格

- **单页面应用**: 垂直布局
- **主要区域**:

- **存储**: localStorage

## 4. 功能规格

### 4.1 私钥管理
- 输入框接受私钥
- 私钥存储在浏览器 localStorage (可选)
- 私钥发送给后端时临时存储在内存中
- UI 上提供停止交易按钮清除内存中的私钥

### 4.2 配置项 (存储于浏览器 localStorage)
- 交易对: btc-updown-5m (固定)
- 限价单价格: 10 (固定)
- 数量: 0.5 (固定)
- 监控提前时间: 30秒 (固定)

### 4.3 交易状态显示
- 当前周期信息
- Yes/No 订单状态
- 成交记录
- 错误信息

### 4.4 日志系统
- 实时显示交易日志
- 时间戳
- 日志级别: INFO, WARNING, ERROR
- 可滚动查看

### 4.5 控制功能
- 开始交易
- 停止交易
- 清空日志

## 5. 后端 API 规格

### 5.1 技术栈
- Python + FastAPI
- 运行端口: 8000

### 5.2 API 端点

#### POST /api/start
- **请求体**: `{ "private_key": "..." }`
- **响应**: `{ "status": "success", "message": "..." }`

#### POST /api/stop
- **响应**: `{ "status": "success", "message": "..." }`

#### GET /api/status
- **响应**: `{ "running": bool, "current_period": {...}, "orders": {...} }`

#### GET /api/logs
- **响应**: `{ "logs": [...] }`

### 5.3 WebSocket
- **端点**: /ws
- **功能**: 实时推送交易状态和日志

## 6. 安全注意事项

- ⚠️ 本地运行，不推荐线上部署
- ⚠️ 私钥仅存储在内存中，程序重启或停止后丢失
- ⚠️ 无权限鉴定，仅供个人使用
